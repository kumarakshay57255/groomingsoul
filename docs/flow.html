01

Public marketing site

The public-facing homepage at /. Hero, trust badges, founder profile, core team, advisory panel, and stats are all rendered from live Postgres data — the admin team can edit any of it from the CMS without code changes.

02

Diploma catalogue (Page 4)

Specialised psychology diplomas, each with strict time-bound access. The dark "Manual Certificate Workflow" section below this listing explains the hardcopy promise to students.

03

Diploma detail page

Curriculum tree, validity badge, instructor, and the prominent "Buy now" CTA. The cert-by-courier promise is highlighted in coral so it can never be missed.

04

Sign-up — Indian mobile validation

When the visitor clicks Buy now while signed out, they're routed through a sign-in / sign-up flow that remembers the course they wanted. The 10-digit Indian mobile validator runs both client- and server-side; the phone is normalised to +91XXXXXXXXXX for storage.

05

Manual QR-payment modal

After signup the student is auto-redirected back to the course with the manual QR checkout modal pre-opened. The official Grooming Souls QR appears on the left, the receipt-upload form on the right with their name + email + phone pre-filled.

06

Receipt screenshot attached

Student picks any payment screenshot from disk. Live preview, file size, and replace button. multer enforces JPG/PNG/WebP only and a 5 MB cap.

07

"Payment under review"

On submit, the purchase row is created with status = pending_verification. The student lands on this confirmation page; nothing is unlocked until an admin verifies the receipt.

08

Admin operations console

Admin sign-in lands here. Realtime KPI tiles — receipts to verify, today's leads, certificates to dispatch, revenue last 30 days. Each clickable tile drills into the relevant queue with the right filter pre-applied.

09

Receipt verification queue

Every Buy Now lands here. Receipts shown as inline thumbnails so the admin can scan down the list at a glance. Status pills filter the view; full-text search across payer name / email / phone / course title.

10

Receipt verification drawer

Click any row → full-size receipt preview, tap-to-call / tap-to-email recipient info, and a free-form admin note. The 90-day validity window comes from the course definition; the admin can't accidentally change it here.

11

Approve & unlock

On Approve, the server sets status = active, stamps activatedAt = NOW(), and computes expiresAt = activatedAt + course.validityDays. The student's dashboard reflects this instantly.

12

Student dashboard (after approval)

The student sees the course as Active with a real-time countdown chip. If less than 7 days remain it flips to Renew soon; minutes-level granularity for last-day cases.

13

Curriculum view

Click Enter course → the full curriculum tree. Each lesson shows duration; completion ticks light up as the student works through the course.

14

Secure video player

HTML5 video served via a signed /api/stream/<JWT> proxy — the real source URL is never sent to the browser. Floating watermark overlays the student's phone + email at random positions every 6.5 seconds. Speed control 0.5×–2×, quality selector (auto-detected resolution), and download/right-click are blocked.

15

Final lesson — cert queued

Marking the final lesson of a diploma auto-inserts a row into certificate_queue server-side. The student sees the green toast confirming their hardcopy certificate is in the dispatch line.

16

Curriculum at 100 %

All three lessons green-ticked, progress bar full. The expiry countdown still runs in the top right — completion doesn't reset the clock.

17

Certificate dispatch queue

Admin → Certificate queue. The student's row appears automatically — no manual data entry. Status pills (Queued · Printed · Dispatched · Delivered) drive both the filter and the per-row workflow.

18

Dispatch drawer

Click the row → drawer with recipient phone + email (copy-to-clipboard), shipping address textarea, courier tracking input, and a Printed → Dispatched → Delivered three-button workflow.

19

Marked Dispatched

Admin fills the address + tracking ID and clicks Dispatched. The server stamps dispatchedAt and saves the tracking ID. In Phase 5 a Resend email and MSG91 SMS will go out to the student with the courier link.
